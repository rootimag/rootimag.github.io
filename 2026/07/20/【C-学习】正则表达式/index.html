正则表达式

1. 正则表达式简介

正则表达式（Regular Expression）是一种用于匹配、查找和替换的工具，以字母、数字和符号组成的微型语言，定义一段“文本模式”。

2. 正则表达式语法

正则表达式由普通字符（如 a, b, c）和元字符（带特殊意义的字符）组成。

2.1 字符类（Character Class）

用于匹配特定范围或类型的单个字符。

语法 含义 示例
. 匹配除换行符以外的任何单个字符 a.c -> abc
\d 匹配一个数字 a\dc -> a1c，等价于 [0-9]
\D 匹配一个非数字 a\Dc -> abc，等价于 [^0-9]
\w 匹配字母、数字、下划线 a\wc -> a1c，等价于 [a-zA-Z0-9_]
\W 匹配非字母、数字、下划线 a\Wc -> a\nc，等价于 [^a-zA-Z0-9_]
\s 匹配任何空白字符 a\sbc -> a bc，等价于 [\t\n\r\f\v]
\S 匹配任何非空白字符 a\Sbc -> abc，等价于 [^\t\n\r\f\v]
[...] 匹配方括号内任意一个字符 a[aeiou]c -> aec、aic
[^...] 匹配不在方括号内任意一个字符 a[^aeiou]c -> abc、adc

2.2 数量词（Quantifier）

用于指定字符的数量。

语法 含义 示例
* 匹配前面的字符0次或多次 a* -> ""、a、"aa"、"aaa"
+ 匹配前面的字符1次或多次 a+ -> a、"aa"、"aaa"
? 匹配前面的字符0次或1次 a? -> ""、"a"
| 匹配左右任意一个表达式 a|b -> "a"、"b"
{n} 精准匹配前面的字符n次 a{2} -> "aa"
{n,} 匹配前面的字符至少n次 a{2,} -> "aa"
{n,m} 匹配前面的字符至少n次，最多m次 a{2,3} -> "aaa"

2.3 边界限定（Anchor）

用于限制匹配发生的位置，不消耗字符。

语法 含义 示例
^ 匹配字符串的开始 ^abc -> "abc"
$ 匹配字符串的结束 abc$ -> "abc"
\b 匹配单词边界（单词与非单词之间） \babc\b -> "abc"
\B 匹配非单词边界（单词与非单词之间） \Babc\B -> "aaabccc"

2.4 贪婪与懒惰匹配（Greedy and Lazy Matching）

用于控制匹配的优先级，正则表达式的数量词（*、+、{}）默认为贪婪匹配，即尽可能多的匹配，懒惰匹配则是尽可能少的匹配。

  • 贪婪匹配：默认值
  • 懒惰匹配：在数量词加上 ?

2.5 分组与逻辑 (Grouping & Logic)

语法 含义 示例
(expression) 匹配括号内的表达式 (a|b)
(?:expression) 匹配括号内的表达式，但不消耗字符 (?:a|b)
\num 匹配第 num 个分组的内容 (a|b)\1

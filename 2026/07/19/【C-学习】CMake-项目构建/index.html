【C++ 学习】CMake 项目构建

学习 CMAKE：从入门到构建 C++ 项目

1. CMake 介绍

在学习 C++ 的过程中，很多人起初只关注语法、算法和数据结构，但是在尝试做一个项目的时候，如何管理越来越多的源文件、头文件、第三方库以及不同平台的编译配置，成了一个棘手的问题。

为了解决这个问题，CMake 被设计，用于管理 C++ 项目的构建过程。

CMake 本身不是编译器，而是一个跨平台的构建系统，根据 CMakeLists.txt 文件中的指令，生成不同平台的编译命令，并执行编译操作，例如：

  • Linux / macOS ：下的 Makefile 和 Ninja
  • Windows：Visual Studio 工程

CMake 核心概念：

  • 工作流：编写 CMakeLists.txt 文件 -> CMake 解析生成底层构建文件 -> 底层工具调用编译器完成编译。
  • 核心思想(Target-Based)：将可执行文件或库视为目标(Target)，所有头文件、编译选项、链接库都严格绑定到 Target。

2. CMakeLists.txt 编写

CMakeLists.txt 文件是 CMake 项目的入口，用于定义项目的构建规则。
以 CMakeLists.txt 为中心，按照项目的生命周期，CMake 指令可分为如下阶段：

2.1 项目初始化

每个 CMakeLists.txt 开头必须包含环境和项目声明：

  1. cmake_minimum_required（设定最低版本）
  • 语法：cmake_minimum_required(VERSION <min>[...<policy_max>])
  • 示例：cmake_minimum_required(VERSION 3.15)
  • 细节：现代 CMake 项目建议要求 3.12 以上版本，以完整支持 Target 属性
  1. project（定义项目属性）
  • 语法：project(<PROJECT_NAME> [VERSION <major>[.<minor>...]] [LANGUAGES <language-name>...])
  • 示例：project(MyProject VERSION 1.0.0 LANGUAGES CXX C)
  • 细节：project 命令会定义 PROJECT_NAME 和 PROJECT_VERSION 变量，便于后续使用
  1. set（定义变量）

2.2 目标构建阶段

决定构建哪些文件：可执行程序、静态库和动态库。

  1. add_executable（定义可执行文件目标）
  • 语法：add_executable(<name> [source1] [source2] [...]])
  • 实例：add_executable(hello src/main.cpp src/core.cpp)
  • 细节：<name> 是此 Target 全局唯一标识符，后续配置均基于此标识符
  1. add_library（定义库文件目标）
  • 语法：add_library(<name> [STATIC | SHARED | MODULE | INTERFACE] [source1] [source2] [...])
  • 参数：
    • STATIC: 静态库（Linux 下为 .a），代码在链接时直接打包进可执行文件中
    • SHARED: 动态库（Linux 下为 .so），运行时动态加载，节省磁盘和内存空间
    • INTERFACE: 纯头文件库（Header-only），不编译源文件，仅传递头文件路径
  • 示例：add_library(mylib STATIC src/mylib.cpp)

Warning: 严禁使用 file(GLOB) 指令，导致 CMake 无法感知文件变化。

2.3 编译期配置阶段

为特定的 Target 指定头文件目录、宏定义和编译器警告选项。

  1. 可见性标识符(Visiblity Specifier)
    所有 Target 配置指令必须携带三个可见标识符之一，用于控制属性的传递

    • PRIVATE：只在当前 Target 内部可见
    • PUBLIC：当前 Target 及其子目标可见
    • INTERFACE：当前 Target 不适用，但子目标可见（常用于纯头文件库）

  2. target_include_directories（定义头文件目录）

  • 语法：target_include_directories(<target> [PRIVATE | PUBLIC | INTERFACE] [prefix1] [prefix2] [...])
  • 示例：target_include_directories(mylib PUBLIC ${PROJECT_SOURCE_DIR}/include)
  1. target_compile_definitions（定义宏定义）
  • 语法：target_compile_definitions(<target> [PRIVATE | PUBLIC | INTERFACE] [macro1] [macro2] [...])
  • 示例：target_compile_definitions(myApp PRIVATE DEBUG=1 USE_NEW_API)
  • 细节：等同于在头文件中 #define DEBUG 1 或者 #define USE_NEW_API
  1. target_compile_options（定义编译选项）
  • 语法：target_compile_options(<target> [PRIVATE | PUBLIC | INTERFACE] [option1] [option2] [...])
  • 示例：target_compile_options(myApp PRIVATE -Wall -Wextra -Werror)

2.4 链接期配置阶段

处理项目对外部库或其他 Target 的依赖关系。

  1. find_package（定义依赖项）
  • 语法：find_package(<package> [version] [EXACT] [QUIET] [MODULE] [REQUIRED] [COMPONENTS [components...]])
  • 参数：
    • EXACT: 指定版本号必须完全匹配
    • REQUIRED: 如果找不到依赖项，CMake 会报错
    • COMPONENTS：指定要使用的依赖项组件
  • 示例：find_package(Boost 1.74 REQUIRED COMPONENTS asio)
  • 细节: 成功后，CMake 会自动提供形如 <PackageName>::<Component> 的导入目标 (Imported Target)。
  1. target_link_libraries（定义依赖项）
  • 语法：target_link_libraries(<target> [PRIVATE | PUBLIC | INTERFACE] [lib1] [lib2] [...])
  • 参数：lib 为依赖项的导入目标，可以为 find_package 找到的依赖项、本项目的其他 Target或系统库名称

  • 细节：链接一个 Target 还会继承其 PUBLIC 和 INTERFACE 属性。

    • FATAL_ERROR: 严重错误（终止配置）

4. CMake Cli 命令

在编写完 CMakeLists.txt 后，我们需要调用 CMake 来构建项目。

现代 CMake 版本提倡树外构建(Out-of-Source Build)，即在项目根目录下创建一个 build 目录，将构建产物放在这个目录下。

4.1 配置阶段(Configure Phase)

解析 CMakeLists.txt 文件，生成底层构建文件（CMakeCache.txt 和 CMakeFiles 目录）。

  • 语法：cmake -S <source_dir> -B <build_dir> [ards...]
  • 参数：
    • -S: 指定源码目录
    • -B: 指定构建目录
    • -G "Generator": 指定构建工具，如 -G Ninja（推荐，快速并行的构建工具）或 -G "Unix Makefiles"（Linux 默认）
    • -DCMAKE_BUILD_TYPE=<type>: 指定构建类型
      • Debug：包含调试符号，无优化（-O0 -g），用于 Debug 模式
      • Release：包含优化，无调试符号（-O2 -DNDEBUG），用于 Release 模式
      • RelWithDebInfo：包含优化，包含调试符号（-O2 -g）

4.2 构建阶段(Build Phase)

使用构建工具调用编译器进行编译。

  • 语法：cmake --build <build_dir> [ards...]
  • 参数：
    • --build build：指定构建目录为 build
    • --parallel <N>（或 -j <N>）：指定并行构建的线程数
    • --target <target>：指定构建的目标
    • --clean-first：先清理构建目录

4.3 清理阶段(Clean Phase)

  • 语法：cmake --build <build_dir> --target clean
  • 作用：删除构建目录下的 .o、.a、.so 及可执行文件，保留 CMake 缓存

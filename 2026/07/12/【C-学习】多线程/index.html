C++ 多线程

1. 多线程概念

在计算机中，多线程涉及以下概念:

  • 进程(Process): 是资源分配的最小单位。
  • 线程(Thread): 是CPU 调度的最小单位。
  • 多线程(Multi-thread): 是多核 CPU 的并行计算。

在硬件条件下，由于单核 CPU 频率难以上升，为了运行效率，需要使用多线程，充分发挥多核 CPU 的能力。

多线程的好处:

  • 程序运行效率提高
  • 避免程序运行阻塞

但是由于异步执行的特性，多线程的缺点:

  • 数据竞争(Data Race)
    当多个线程同时访问同一个数据时，可能会导致数据不一致的情况出现。
  • 竞态条件(Race Condition)
    程序执行的顺序是不确定的，可能会导致有依赖关系的的代码被随机错误地执行，极难调试。
  • 死锁(Deadlock)
    当多个线程互相等待对方的资源，造成互相等待，导致死锁。
  • 性能下降
    多线程的创建会导致线程切换的开销，使用不当降低了程序的性能。

2. 多线程的实现

C++ 在 C++11 引入了 <thread> 线程库，使得我们可以忽略不同平台的底层线程 API 差异。

实现细节:

  • class std::thread 为线程的封装类，包含了线程的生命周期管理等功能。
    维护以下内容:
    • 线程句柄(Thread Handle): 操作系统中的线程标识符，用于 join() 和 detach()。
    • 线程栈(Thread Stack): 操作系统分配的私有栈空间，用于存储线程的本地数据（可用 ulimit -s 查看大小）。
    • TLS(Thread Local Storage): 线程局部存储，用于存储线程的本地数据。

Warning: 线程不可复制，即 thread(const thread&) = delete;，但可以使用移动语义

底层操作:

  • 类型擦除(Type Erasure): std::thread 为了接受所有可调用对象(lambda，函数对象，函数指针等)，会打包进 std::function 对象。
  • 构造函数拷贝/移动参数: 正如上文，线程有私有的线程栈，需要将参数拷贝或移动到线程栈中（主要性能开销）。
  • 异常捕获: 底层入口函数会包裹一个 try-catch 块，用于调用 std::terminate()，防止程序崩溃。

C++ 在 C++20 引入了 std::jthread ，可在析构时自动调用 join()，并且实现了协作式中断(Stop Token)，通过 std::stop_token 检查是否停止。

3. 多线程的使用

  1. 创建线程(<thread>)
  • 语法: std::thread t(func, args...);
  • 参数:
    • func: 要执行的可调用对象。
    • args...: 可调用对象的参数。
  • 细节:
    • 引用传递: 可以使用 std::ref() （位于 <functional>） 函数将对象引用传递给线程，优化性能。
    • 生命周期: 创建后必须使用 t.join() （阻塞主线程直到结束） 或 t.detach() （分离线程后台运行），优先使用 std::jthread, t.join() 或 RAII 封装。
  1. 互斥量(<mutex>)

解决线程间的数据竞争问题。

  • 语法: std::mutex mtx;
  • 细节:
    • mtx.lock() 和 mtx.unlock() 用于加锁和解锁（Warning: 不建议使用）。
    • std::lock_guard<std::mutex> lock(mtx);: RAII 的典范，用于自动加锁和解锁，在多个互斥量情况下容易死锁（推荐使用）。
    • std::unique_lock<std::mutex> lock(mtx);: 给单个互斥量加锁，防止死锁（推荐使用）。
    • std::scoped_lock<std::mutex> lock(mtx);: 给多个互斥量加锁，防止死锁（推荐使用）。
  1. 原子操作(<atomic>)

对互斥量的优化。

  • 语法: std::atomic<T> var;
  • 细节:
    • var.load() 读取当前变量值。
    • var.store(x) 设置变量值为 x。
    • 保证数据的原子性，无须加锁，性能更好。

atomic_flag 优化

std::atomic_flag 是 C++ 标准中唯一保证总是无锁(Lock-Free)的原子类型，不支持 load 和 store 操作，但可以使用 clear （清除为 false） 和 test_and_set （设置并返回旧值，设置为 true）和 test() （C++ 20, 返回当前值） 操作来实现原子操作，用于构建高性能的自旋锁(Spinlock)。

自旋锁(Spinlock): 自旋(Spin)的线程不会进入休眠状态，而是进行空循环(Busy Waiting)，相比于互斥量(Mutex)，不涉及上下文切换(Context Switch)，在锁释放的极快时，自旋消耗的 CPU 可能远小于上下文切换的成本。

为了解决此类问题，C++ 的答案为 Compare-and-Swap(CAS) 操作，提供了一个语义：“如果值等于预期值，就把它改成新值；否则就什么都不做。”。

C++ 定义了 std::atomic::compare_exchange_strong() 和 std::atomic::compare_exchange_weak() 函数实现，之所以有两个函数，是因为strong 严格保证原子性，而 weak 不严格，可能出现伪失败(Spurious Failure)。

在 x86/x64 (CAS 架构) 上，提供了 CPU 指令 LOCK CMPXCHG，天然强指令，故 strong 和 weak 开销基本一致，而在 ARM/PowerPC/RISC-V (LL/SC 架构) 上，CPU 不提供 CMPXCHG，而是使用 LL/SC(Load-Link/Store-Conditional)，操作是分离的，可能出现判断成功，但操作失败的情况，在此情况下 weak 性能强于 strong。

特性 strong weak
原子性 严格保证 不严格，可能出现伪失败
开销 可能略慢 在 ARM 等架构更好
场景 单次逻辑判断 循环无锁判断

语法: var.compare_exchange_{type}(expected, desired)

  • expected: 预期值
  • desired: 新值

内存序(Memory Order) 优化

默认情况下，原子操作均使用 std::memory_order_seq_cst （顺序一致性），保证全局执行顺序。

  • std::memory_order_relaxed: 仅保证原子性，不保证指令顺序，编译器会乱序优化。
  • std::memory_order_acquire / release: 建立同步关系。Release 保证此操作之前的所有内存写入对执行 Acquire 的线程在该操作之后可见，建立先行发生关系(Happens-Before)。
  • std::memory_order_seq_cst: 默认，最严格，保证指令顺序，但性能开销大。
  • std::memory_order_acq_rel: 用于**读-改-写(Read-Modify-Write)**操作，在读取时为 acquire，写入时为 release。

类似 compare_exchange_weak ，修改内存序对 ARM 等架构优化较大，对 x86/x64 影响较小。

  1. 线程间通信(Inter-Thread Communication)

在多线程中，为了等待特定条件，需要使用 std::condition_variable，严禁使用 while。

条件变量(condition_variable)

用于线程挂起与唤醒，实现线程同步。

  • 语法: std::condition_variable cv;
  • 依赖: 必须配合 std::unique_lock<std::mutex> 使用。
  • 调用:
    • cv.wait(lock, predicate);: 若不满足条件则挂起并释放锁，直到满足条件为止。
    • cv.notify_one(); / cv.notify_all();: 唤醒一个或所有线程。

Warning: 操作系统可能在无信号条件下唤醒线程，建议 wait 搭配 lambda 谓词 使用。

4. 线程辅助库(<this_thread>)

  1. std::this_thread::sleep_for
  • 语法: std::this_thread::sleep_for(std::chrono::duration<Rep, Period>);
  • 参数:
    • std::chrono::duration<Rep, Period>: 时长，Rep 是算数类型，Period 是时长单位，为 std::ratio<Num, Den 类型，表示每 Num 秒有 Den 个单位。常用的 milliseconds, seconds 都是 Duration 的别名(alias)。
  • 作用: 让出 CPU 时间片，挂起线程，直到时间耗尽再被唤醒。
  1. std::this_thread::yield
  • 语法: std::this_thread::yield();
  • 作用: 主动让出 CPU 时间片，挂起线程，直到下次被调度，通常与自旋锁一起使用。
  1. std::this_thread::get_id
  • 语法: std::this_thread::get_id();
  • 作用: 获取当前线程 ID，方便调试。
